HTML Editors

The 2 html editors that we will be using in the class are HTML Code Editor and Online HTML Editor along with a Loripsum Generator. The first editor allows you to write your document in HTML in the left pane and see the result in the right pane. The second editor allows you to edit a document in with a word like word processor and see the HTML result in the right pane. You can also make modifications on the right pane and have them reflected in the left pane.

CSS (Cascading Style Sheets)

Definition

Cascading Style Sheets is a style sheet language that describes how HTML elements are to be displayed in an HTML document. A cascading style sheet is normally placed in a seperate file with a css extension.

When styling HTMLelements the concept of the box model comes into play. CSS Box Model

History

The initial release was December 17, 1996 and was developed by H√•kon Wium Lie, Bert Bos and the World Wide Web Consortium (W3C).

Appearence

CSS to style a simple heading element with a color of red and text centered would look like this:

h3 {
  color: red;
  text-align: center;
}
CSS Selectors

CSS selectors enable us to select an element for styling. For the class the main ones we will be working with are the element selector, element.class selector and element element selector. The element selector selects all elements with that tag name. For example p selects all <p> elements for styling. The element.class selector selects all elements with that class name. For example p.end selects all <p class="end"> elements for styling. The element element selector selects all elements that have a parent as the first element. For example nav a selects all <a> elements that are inside of a <nav> element.

Multi-Join Example
			    SELECT Customers.CustomerName AS Customer,
       Orders.OrderID AS OrderID,
       Employees.LastName AS Employee,
       Products.ProductName AS Product,
       Categories.CategoryName AS Category,
       Suppliers.SupplierName AS Supplier,
       Shippers.ShipperName AS Shipper
FROM (((((((Orders INNER JOIN Customers ON Orders.CustomerID = Customers.CustomerID)
INNER JOIN Shippers ON Orders.ShipperID = Shippers.ShipperID)
INNER JOIN OrderDetails ON Orders.OrderID = OrderDetails.OrderID)
INNER JOIN Products ON OrderDetails.ProductID = Products.ProductID)
INNER JOIN Suppliers ON Products.SupplierID = Suppliers.SupplierID)
INNER JOIN Categories ON Products.CategoryID = Categories.CategoryID)
INNER JOIN Employees ON Employees.EmployeeID = Orders.EmployeeID)
ORDER BY Customers.CustomerName, Orders.OrderID, Products.ProductName;

API's

Definition

API stands for Application Program Interface. An API describes clear methods of communication between software components. An API enables you to talk to other software easily and to build software applications more easily. An API might describe rules for using a library. A Web API would allow applications to communicate via the web. You can find a list of simple public web API's on github.Public web API's

An analogy of an API might be a bank teller. Interactions with the bank occur with the teller or online. The teller becomes the interface to obtain service from the bank. You are not allowed to go beyond the teller station and conduct business yourself. The bank can change tellers, equipment, etc... without altering the service they are providing to you. You are still allowed to make deposits and withdrawls and they can decide the method or methods to implement deposits and withdrawls.

Advantages

Part of an application could be outsourced to an API. For example a map API could be used instead of developing the code yourself.
