HISTORY/ORIGIN -

  • CSS - The initial release was December 17, 1996 and was developed by Håkon Wium Lie, Bert Bos and the World Wide Web Consortium (W3C).

CSS

  • CSS Cascading Style Sheets.
  • What is is - A language used to design the layout of a web page. It was developed to allow separation of presentation (format) and content. You could imagine the headaches involved if you had to style each HTML element individually. With CSS we can place the styling rules in a separate file.
  • Cascading - multiple style sheets can be applied to the same page.
  • Developer Console - If you have Chrome or Firefox you can see the style rules that are being applied to the page. For FireFox choose the style editor. (Chrome CTRL-SHIFT-I, Firefox CTRL-SHIFT-K).

CSS Syntax -    The format of the CSS rule will generally be with the selector on the far left followed by a curly brace then the styling property followed by a colon and then the value.

Generally when multiple rules need to be applied each rule is listed on a separate line.

For example if you wanted to style a heading element with the color of red then the format would look like this:

h1 { color: red }

If multiple rules need to be applied then a semicolon would be used after the style value followed by the next style rule.

In this example the same element would be used but an additional style rule would be applied.

h1 { color: white;

        background-color: black;

}

CSS Selectors - are used to select elements for styling. In the previous example the element selector was used. There are 2 other simple selectors that are commonly used.  They are the class selector and the id selector.

The class selector allows us to style elements who have a class attribute set. For Example to style all h1 elements that have the class of heading the syntax would look like:

.heading { color: red }

The corresponding html element would look like:

<h1 class=”heading”>Heading</h1>

The id selector allows us to style elements who have an id attribute set. For example to style the h1 element that has an id of main-heading the syntax would look like:

#main-heading { color: red }

The corresponding html element would look like:

<h1 id=”main-heading”>Heading</h1>

Applying The CSS Rules: We can apply the style sheet rules in 3 different ways.

  1. External Style Sheet - With this method all of the style rules are kept in a separate file and the main html page is linked to it with a link element in the head section of the page.

For Example the following represents an html page with a link to a style sheet called “my-style.css”

<!DOCTYPE html>

<html lang=”en”>

<head>

<link rel="stylesheet" type="text/css" href="my-style.css">

</head>

<body>

<h1>Your Heading</h1>

<p>Your Paragraph Content.</p>

</body>

</html>

This is the most common but in this class we are probably going to be using internal style.

  1. Internal Style Sheet - with this method all of the style rules are placed between an opening and closing style tag in the head section.

        

        For Example the following represents an html page with internal style.

<!DOCTYPE html>

<html lang=”en”>

<head>

<style>

h1 {

  color: green;

}

</style>

</head>

<body>

<h1>Your Heading</h1>

<p>Your Paragraph Content.</p>

</body>

</html>

  1. Inline - Style is applied to the element itself through a style attribute.

For example the following html page uses inline styling to style the heading element and the paragraph element:

<!DOCTYPE html>

<html lang=”en”>

<body>

<h1 style="color:red; text-align:center;">Your Heading</h1>

<p style="color:blue;">Your Paragraph Content.</p>

</body>

</html>

This type of style is not used very often in commercial web pages due to the complexity of maintaining the style for every single element. However for test purposes this is fine.
